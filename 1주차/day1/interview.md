## 1. 클로저란 무엇인가?

### 내 답변

- 클로저는 함수가 선언될때 외부의 스코프를 기억하고 변수를 참조가능한 상태를 말합니다, 해당 스코프는
[[environment]] 라는 슬롯에 저장되며 참조에 사용됩니다. 클로저는 주로 정보은닉, 콜백 처리, 상태 유지 같은 상황에서 사용됩니다.

### 체크 포인트

- 함수가 선언될 때의 바깥 스코프를 기억한다는 점
- 함수가 나중에도 외부 변수를 참조할 수 있다는 점
- 상태 유지, 콜백, 정보 은닉 같은 활용 예시

## 2. `var`와 `let`은 반복문에서 왜 다른 결과를 만들 수 있는가?

### 내 답변

- `var`의 경우 함수 단위의 스코프를 갖기 때문에 반복문 내부에서 비동기 콜백이 `var`로 선언된 변수를 참조하면, 반복문이 끝난 뒤의 동일한 변수 값을 참조할 수 있다. 즉, 여러 콜백이 같은 변수를 공유해서 같은 값만 참조하는 상황이 생길 수 있다.
- 반면 `let`은 블록 스코프를 가지며, `for`문에서는 매 반복마다 새로운 바인딩이 만들어진다. 그래서 각 반복에서 만들어진 콜백이 서로 다른 반복 시점의 값을 참조할 수 있다.

### 체크 포인트

- `var`는 함수 스코프, `let`은 블록 스코프라는 점
- 반복문에서 `let`은 각 반복마다 새로운 바인딩이 만들어질 수 있다는 점
- 비동기 콜백과 함께 쓸 때 차이가 더 잘 드러난다는 점

## 3. React에서 stale closure는 언제 발생하는가?

### 내 답변

- stale closure는 예전 렌더에서 오래된 상태값을 참조하는 경우 발생할 수 있다. 예를 들면 setInterval 코드를 useEffect 내부에서 구현하고 1초마다 count를 증가시키는 타이머를 구현한다고 예를 들었을때 만약 setCount(count + 1) 이런식으로 구현한다면 해당 코드에선 count가 첫 렌더당시의 값을 계속 참조하게 되므로 초기값 + 1 의 값만 계속 출력하게 된다 이러한 경우를 stale closure 라고 부른다. 이는 함수형 업데이트나 의존성 배열로 해결이 가능하다.

- 또한 이전 상태를 기준으로 다음 상태만 계산하면 되는 경우에는 함수형 업데이트를 활용하는 것이 좋다. 함수형 업데이트는 React가 넘겨주는 최신 이전 상태를 기준으로 계산하므로 오래된 클로저 문제를 피할 수 있다. 반면 Effect가 특정 상태값을 사용해서 API 요청, 구독 대상 변경, 이벤트 리스너 등록 같은 외부 작업을 다시 동기화해야 한다면 의존성 배열을 사용하는 것이 좋다. 다만 단순 누적 업데이트만 필요한 상황에서 의존성 배열에 상태를 넣으면 Effect가 상태 변경마다 정리되고 다시 실행될 수 있으므로 함수형 업데이트가 더 적절한 경우가 많다.

### 체크 포인트

- 예전 렌더에서 만들어진 함수가 오래된 상태값을 참조하는 상황
- `setInterval`, `setTimeout`, 이벤트 리스너, 비동기 콜백 예시
- 함수형 업데이트나 의존성 배열로 해결할 수 있다는 점

## 4. 렉시컬 스코프와 스코프 체인은 무엇이며 어떤 관계가 있는가?

### 내 답변

- 렉시컬 스코프는 함수가 선언된 위치를 기준으로 상위 스코프가 결정되는 것을 말하고, 스코프 체인은 식별자를 참조할 때 현재 스코프에 값이 없으면 상위 스코프로 탐색을 이어나가는 구조를 말한다. 즉, 렉시컬 스코프가 어떤 상위 스코프를 가질지 결정하고, 스코프 체인은 그 구조를 따라 식별자를 탐색한다.

### 체크 포인트

- 렉시컬 스코프: 선언 위치 기준으로 상위 스코프가 결정되는 규칙
- 스코프 체인: 변수를 찾을 때 현재 스코프에서 바깥으로 올라가며 탐색하는 구조
- 선언 기준과 탐색 방식의 차이를 구분해서 설명할 수 있는지

## 5. 호이스팅과 TDZ를 함께 설명해보세요.

### 내 답변

- 호이스팅은 JS 엔진이 코드를 실행하기 전에 선언 정보를 먼저 처리하기 때문에, 선언문이 위로 끌어올려진 것처럼 보이는 현상을 말한다. `var`, `let`, `const`는 모두 호이스팅되지만 초기화 시점이 다르다. `var`는 선언과 함께 `undefined`로 초기화되므로 선언 전에 접근하면 `undefined`가 출력된다. 반면 `let`, `const`는 선언은 먼저 처리되지만 실제 선언문을 만나기 전까지 초기화되지 않기 때문에, 초기화 전에 접근하면 `ReferenceError`가 발생한다. 이 초기화 전 접근 불가 구간을 TDZ라고 한다.

- 함수 선언문은 실행 전에 함수 이름과 함수 객체가 함께 준비되므로 선언 전에 호출할 수 있다. 반면 함수 표현식은 함수 자체가 미리 준비되는 것이 아니라, 함수를 담는 변수 선언 방식에 따라 호이스팅된다. `var` 함수 표현식은 할당 전에 호출하면 `undefined`를 함수처럼 호출하게 되어 `TypeError`가 발생하고, `let`/`const` 함수 표현식은 TDZ 때문에 선언 전에 접근하면 `ReferenceError`가 발생한다.

### 체크 포인트

- `var`, `let`, `const` 모두 호이스팅된다는 점
- `var`는 `undefined`로 초기화되지만 `let`, `const`는 초기화 전 접근이 막힌다는 점
- TDZ는 초기화되기 전까지 접근할 수 없는 구간이라는 점
