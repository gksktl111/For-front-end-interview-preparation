# Frontend Foundations Bootcamp

프론트엔드 개발에 필요한 핵심 개념을 30일 동안 체계적으로 학습하고, 실제 프로젝트와 연결해 정리하는 학습 저장소입니다.

단순히 기술의 사용법을 익히는 데 그치지 않고 다음 내용을 설명할 수 있는 수준을 목표로 합니다.

- 각 기술이 어떤 문제를 해결하는가
- 기술이 내부적으로 어떻게 동작하는가
- 어떤 기준으로 기술과 구조를 선택해야 하는가
- 선택 가능한 다른 방법에는 무엇이 있는가
- 실제 프로젝트에서 어떻게 적용할 수 있는가

## 학습 목표

이 저장소는 다음 역량을 강화하기 위해 만들어졌습니다.

- JavaScript와 TypeScript의 핵심 동작 원리 이해
- 브라우저, 네트워크, HTTP 동작 이해
- React의 렌더링과 상태 관리 원리 이해
- Next.js의 서버와 클라이언트 실행 경계 이해
- 상태의 성격에 따른 적절한 관리 방법 선택
- 테스트 전략과 웹 접근성 학습
- 성능 문제 측정과 원인 분석
- 책임과 의존성을 기준으로 한 아키텍처 설계
- 빌드와 배포 과정 이해
- 배포된 프로덕트의 운영과 관측성 이해
- 기술 선택 이유와 트레이드오프 설명 연습

## 진행 방식

매일 하나의 주제를 중심으로 다음 과정을 반복합니다.

1. 핵심 개념을 학습합니다.
2. 작은 코드 실험을 실행합니다.
3. 기존 프로젝트에서 관련 사례를 찾습니다.
4. 현재 구현의 장점과 문제점을 분석합니다.
5. 면접 질문과 답변을 작성합니다.
6. 이해하지 못한 내용을 다음 복습 항목으로 남깁니다.

전체 학습 흐름은 다음과 같습니다.

```text
개념 학습
→ 코드 실험
→ 프로젝트 사례 분석
→ 적용 여부 판단
→ 면접 답변 작성
→ 복습
```

## 학습 구성

### 1주차

JavaScript, TypeScript, 브라우저와 네트워크의 기본 동작을 학습합니다.

주요 내용은 다음과 같습니다.

- JavaScript 실행 환경
- Scope와 Closure
- Event Loop와 비동기 처리
- 객체와 참조 동일성
- TypeScript 타입 시스템
- Narrowing과 Union
- 브라우저 렌더링 과정
- HTTP와 네트워크
- Cookie, CORS, Cache

### 2주차

React의 렌더링 원리와 컴포넌트 설계를 학습합니다.

주요 내용은 다음과 같습니다.

- Render와 Commit
- State와 Snapshot
- Effect
- Ref
- Controlled와 Uncontrolled
- Form 상태
- 상태 끌어올리기
- Composition
- Custom Hook
- 웹 접근성
- 컴포넌트 테스트

### 3주차

Next.js App Router와 애플리케이션 상태 관리를 학습합니다.

주요 내용은 다음과 같습니다.

- App Router
- Server Component
- Client Component
- Hydration
- 서버 데이터 패칭
- 캐시와 재검증
- URL 상태
- 인증과 권한
- TanStack Query
- Zustand
- 상태 소유권

### 4주차

성능, 아키텍처, 테스트, 빌드, 배포와 프로덕션 운영을 학습합니다.

주요 내용은 다음과 같습니다.

- 성능 측정 도구
- Web Vitals
- 네트워크와 번들 최적화
- 프론트엔드 아키텍처
- 오류 처리와 확장성
- 테스트 전략
- Production Build
- 정적 배포와 서버 배포
- CI와 CD
- AWS S3와 CloudFront
- 프로덕션 운영
- 로그, 메트릭, 트레이스
- 장애 대응과 Rollback

## 학습 원칙

- 개념을 외우기보다 동작 원리를 이해합니다.
- 라이브러리와 프레임워크를 정답이 아닌 선택지로 봅니다.
- 성능 개선은 측정 후 진행합니다.
- 구조 변경은 파일 크기가 아니라 책임과 변경 이유를 기준으로 판단합니다.
- 프로젝트 경험과 일반적인 기술 개념을 구분합니다.
- 기술 선택의 장점뿐 아니라 단점과 전환 조건도 기록합니다.
- 실제로 검증하지 않은 결과를 단정하지 않습니다.
- 학습한 내용을 자신의 언어로 설명합니다.
- 모르는 내용은 추측하지 않고 검증 방법을 찾습니다.


## 빌드와 배포 학습 원칙

빌드와 배포는 특정 플랫폼의 사용법만 익히지 않고 전체 흐름을 이해하는 것을 목표로 합니다.

```text
소스 코드
→ 의존성 설치
→ 정적 분석과 테스트
→ Production Build
→ Artifact 생성
→ 대상 환경으로 전달
→ 새 버전 공개
```

다음 배포 방식을 비교합니다.

- 정적 호스팅
- Node.js 서버 배포
- Container 배포
- Serverless 배포
- 관리형 플랫폼 배포
- 직접 운영 방식

AWS S3와 CloudFront를 활용한 정적 배포와 관리형 플랫폼 배포를 비교하며, 각 방식에서 직접 관리해야 하는 책임을 확인합니다.

## 프로덕션 운영 학습 원칙

프로덕션 운영 학습은 배포가 완료된 상태에서 시작합니다.

```text
배포 완료
→ 사용자 트래픽 유입
→ 상태 관측
→ 이상 탐지
→ 원인 분석
→ 복구
→ 재발 방지
```

다음 내용을 중심으로 학습합니다.

- Process와 Container 상태
- Health Check
- Client Error
- Server Log
- Metric
- Trace
- Alert
- Release Tracking
- Rollback
- 장애 대응
- Runbook
